Interaction: SPB vs. Gatekeeper

There are two ways in which SPB and Gatekeeper work together:

  1. SPB provides the whitelist file for Gatekeeper.
    The SPB API provides a rest call that sends a properly formatted whitelist file in its response. On the gatekeeper server, a cron job is configured that pulls this whitelist file from Gatekeeper at regular intervals (e.g. every 20 minutes). Gatekeeper will automatically reload this whitelist file when it detects a change.
    If SPB is not accessible, the cron job will not update the whitelist file, and Gatekeeper will continue to use the last whitelist file it managed to pull from SPB.
  2. Gatekeeper sends log messages to SPB.
    This makes it possible to view the Gatekeeper logs through the SPB UI.
